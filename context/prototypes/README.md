# Design Prototypes (designer agent)

Mockups produced by the **designer** agent for review before any production code is
written. One **numbered sub-folder per iteration** (`001-`, `002-`, …) — never
overwrite or renumber; a new direction is always a new folder. Each folder holds
self-contained HTML (openable by double-click) plus a `NOTES.md`.

These are **proposals**, not shipping code. An accepted direction becomes a feature
spec, then the `nextjs-ui-builder` (public pages) or `pdf-render-specialist` (print)
builds it for real.

## Index

- [001 — Public pages refresh](001-public-pages-refresh/index.html) — full public set (landing, catalog, detail, order, policies, download) on the existing design system; introduces a celebrate/remember "two worlds" split of the 8-title catalog
- [002 — Book detail: gallery + questions](002-book-detail-gallery-questions/index.html) — single-image gallery viewer → one-at-a-time carousel lightbox (prev/next + counter, scales 2→13 images), a full grouped "Questions you'll answer" section, and a neutrally-framed worked-example panel of the fixture answers behind each sample (now incl. the real uploaded **source photo** in a polaroid treatment — "this photo → this book"); 3 variants (Story 1 image-heavy/rose · Story 7 living/gold · Story 5 sparse-pair/rose)
- [003 — Logo concepts](003-logo-concepts/index.html) — 5 logo directions to replace the plain text header wordmark, each in hero lockup + live nav + light/gold/rose surfaces + favicon scale; recommends **02 "The Bound Spine"** (stitched book-spine glyph, names the product + puns "-bound", neutral across both worlds)
