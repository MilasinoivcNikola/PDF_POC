## 2026-06-09 — Preview ↔ PDF Image Parity & True-Scale Preview

**Branch:** `fix/preview-pdf-image-parity` → `main` (`5caf5ed`, merge `623d27a`) · Craft Area 1 (PDF) + 3 (preview UI)

Milestone 6 polish. Started as a two-defect bug report (the in-browser preview and the downloaded PDF disagreed on illustration size/crop) and grew, through PM iteration, into a faithful WYSIWYG preview. Five related fixes, all CSS / preview-rendering — no story copy, merge/variant, or generation-pipeline change. The hard invariants held throughout: **exactly 14 Letter pages**, no text clipping, the "died" rule, and **screen↔PDF parity** (every changed selector mirrored in both `lib/pdf/styles.css` and `app/globals.css`, tokens single-sourced via `var()`).

- **PDF crop parity.** Root cause: source illustrations are square (1024×1024) shown via `object-fit: cover`, but the print landscape slot was 7.0in × 3.2in = **2.19:1** (a wide letterbox that cut the pet's body off) vs the screen's 4:3 — the two render targets disagreed on slot ratio. Made the print slot a centered box matching the preview crop. (`lib/pdf/styles.css`)
- **Page 7 art.** The "gentle truth" page (`TruthPage`) used a bespoke ~⅓-size thumbnail. Swapped its art to the shared `story-art story-art--landscape` slot (`lib/pdf/pages.tsx`) so it matches the other pages, keeping the centered "quiet" death-page text.
- **Preview image proportion.** Screen landscape art was full-column-width while the PDF's was a centered fraction; set the screen art to the print proportion so image-to-page matched.
- **True-scale preview (the core change).** Diagnosed the deeper issue: the preview was **not a scale model of the PDF** — body text is the same 16.8px (`1.05rem`, `html{font-size:16px}`) in both, but the preview page was only ~65% of the print page, so the same-px text read ~1.5× larger relative to the page (and the matched image exposed it). Fix (PM chose it over per-element font patching): each preview `.story-page` now renders at the PDF's **real inch geometry** (8.5×11in, 0.75in padding, print font/spacing) and is scaled to fit its column via `zoom: var(--preview-scale)`, with a `ResizeObserver` in `components/preview/BookPreview.tsx` setting `--preview-scale = cellWidth/816`. Bespoke per-page fonts (cover/dedication/love/closing/truth) aligned to the print fixed values, scoped under `.preview-page`. Controls/editor stay full-size as siblings outside the zoomed page. Measured parity: text÷page, image÷page and padding all match print within ~1%.
- **Larger illustrations + page-7 top spacing (PM request).** Bumped `.story-art--landscape` to **6.27×5.2in** (≈6:5, ~74% of page width) and changed `.story-page--truth` to `justify-content: flex-start` so page 7's image starts at the same top spacing as the others, in both targets.

**Height-safety (the +2in image risk).** A 5.2in image (up from 3.2in) could overflow the text-heavy pages. Rendered the real Otis book **and** a worst-case variant (age 9-12 + euthanasia + other-pets — the max-paragraph case) and measured every page: **no text clipping, exactly 14 pages** in both. On the single most text-heavy page (page-11) of the extreme variant the flex layout shrinks that one image to ~4.56in to preserve the text (content-safe trade); the real book renders every image at a uniform 5.2in.
