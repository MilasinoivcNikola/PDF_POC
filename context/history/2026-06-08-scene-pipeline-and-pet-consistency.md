Milestone 3 — **completes Milestone 3 (the "wow moment")**. Closed the loop on the project's central craft problem: from one pet photo, produce a whole Story-1 book of illustrations where the pet is recognizably the *same animal* on every page, assemble them into a real PDF, and make re-runs cheap. Extends feature 06's `lib/ai/` surface; adds three modules + four test files.

- `lib/ai/prompts.ts` — per-scene prompt builders driven by the **resolved per-page illustration briefs** (`resolveStory` from feature 03), so art and text stay in sync with no duplicated scene copy. `SCENE_PAGE_IDS` = the 13 illustrated slots (cover + page-1…12; back-cover excluded as a writing page) matching `prototypes/generating.html`. Each scene wraps its brief in "same animal as the references" framing + a single-sourced style/consistency clause (warm pastels, golden-hour, no pure black, identical breed markings / coat / eye color / posture, child rendered 3/4-or-from-behind). Sparse session → `MergeError`, never a surviving `{placeholder}`.
- `lib/ai/generate.ts` — adds `generateSceneIllustration()` (low-level, enforces the 16-image cap as a final guard), the orchestrator `generateAllIllustrations(session, options)` (reference → all scenes → saved PNGs under `./generated/[id]/[page].png` → `GeneratedImage[]` manifest with `promptHash`+`referenceHash`), `regenerateSceneIllustration()` (single-page path for feature 10), and `manifestToImageMap()` (manifest → `PageImageMap` of self-contained `data:` URLs for feature 05's renderer). Feature 06's reference function untouched.
- **Consistency strategy is configurable, not hard-wired** (`ConsistencyApproach`): **A** (default, `[photo + reference]` per scene, scenes independent ⇒ parallel), **B** (accumulate prior scenes as refs, capped at 16, sequential), **C** (photo-only baseline). Every disk write goes through the `isSafeSessionId`/`resolveUnder` traversal guards; the photo read is contained under `./uploads`.
- `lib/ai/cache.ts` — `hashPrompt` / `hashReferenceSet` (SHA-256, order-sensitive + length-prefixed so Approach B's accumulating sets can't collide by re-segmentation); `findCachedImage` is a hit only on matching page + both hashes **and** the PNG still on disk ⇒ a re-run is a pure lookup, zero spend. Quality tiers honored: reference `low`, scenes `medium` default, `high` reserved.
- `lib/ai/retry.ts` (new, **found and fixed during live QA**) — the mocked unit tests were green, but the first real run **429'd**: the account allows only ~5 image-input requests/min and the orchestrator fired all 13 Approach-A scenes through an unbounded `Promise.all`. Added `mapWithConcurrency` (cap 3, input-order-preserving) + `withRetry` (retries true 429s only, honors the server's `Retry-After`/"try again in Ns" hint, exponential floor + jitter, cap 5; fails fast on any other error). Public signatures and the cold-success "one API call per image" count unchanged.
