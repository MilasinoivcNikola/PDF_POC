## 2026-06-11 — Commerce PR-08: Admin Review & Approval (auth-gated)

**Branch:** `feature/admin-review` → `main` (`5540b7a`, merge `ca50124`) · Phase 3b (nextjs-ui-builder)

Commerce build, PR-08 — **Phase 3b, the operator approval gate.** An auth-gated admin where the operator reviews each generated book, repaints any drifted page, and **Approves** → final PDF in Supabase. The ~30-second human glance that keeps a grief product from shipping a book where the pet drifted. **This PR introduces the first authentication in the entire codebase** (Supabase Auth — the roadmap's noted default, PM-confirmed). It reuses the whole preview/repaint/edit/render-pdf stack unchanged; the new surface is the auth gate + the admin pages + the approve/requeue routes. Story-1 and Story-2 orders both review/approve identically (registry-driven render). Closes the commerce-roadmap Phase 3 approval gate.

- **The auth gate (the headline + the security-critical surface).** `lib/supabase/auth.ts` (new) — a **second** Supabase client, kept strictly separate from the existing service-role one: cookie-based session via **`@supabase/ssr`** using the **anon** key (RLS-bound, gate-only). `getOperatorUserId()` calls `auth.getUser()`, which **re-validates the token server-side** (not the trust-the-cookie `getSession()`), so a tampered/expired cookie can't pass. Server-only (touches `next/headers`), never a client bundle. `app/(operator)/api/admin/auth/route.ts` — `POST` sign-in / `DELETE` sign-out; bad creds return an **opaque `invalid_credentials`** (never echoes the provider error, the email, or which field was wrong). **Dual gate** on the admin **pages** (`getOperatorUserId()` → `redirect("/admin/login")`) and the new **mutation** routes (`assertOperator()` as the first statement → 404 under public, then the session check → 401).
- **The queue + review + approve.** `app/(operator)/admin/page.tsx` — auth-gated queue of `awaiting_review` (linked to detail) + `failed` (stored error + a **Re-queue** button → `failed → queued`). `app/(operator)/admin/[orderId]/page.tsx` + `AdminBookReview.tsx` — reuses `BookPreview` (preview + per-page regenerate + inline text edit) against the order's on-disk book. **The PR-07 bridge:** the worker writes each generated book to local disk keyed by the **order id** (`./sessions/[orderId].json` + `./generated/[orderId]/`), so `orderId === sessionId` locally and the whole preview/repaint/edit stack works **unforked** — `BookPreview` just took an explicit `sessionId` prop + an optional `renderActions` slot (the wizard caller omits both → **byte-for-byte unchanged**). Approve rides the existing `busy = downloading || regenerating || savingText` race guard, so it can't capture a mid-repaint book. `app/(operator)/api/admin/approve/route.ts` — `manifestToImageMap` → `renderStoryPdf` → `putPdf(order-pdfs)` → `updateOrderStatus(orderId, "approved", { pdfKey })`. Approve renders a PDF (Puppeteer) but **generates nothing** (no OpenAI call) — the spend guard is intact.
- **Store change (minimal):** `lib/order/store.ts` `updateOrderStatus` gained an optional `pdfKey`, written in the **same** state-machine-guarded patch as the status (after `assertTransition`), so an order is never `approved` without its `pdfKey` and an illegal move never writes a stray key (mirrors the existing `error`-on-`failed` pattern — no new function, no forked mapping).
- **The documented reused-route boundary (the call the security review owns).** The admin detail view reuses the existing operator engine routes (`/api/preview`, `/api/regenerate-illustration`, `/api/update-text`, `/api/render-pdf`), which are gated **only** by `assertOperator()` (no session) because they're shared with the non-authed operator wizard. So on the operator's localhost a party with local access could hit `/api/regenerate-illustration` directly and spend, or `/api/preview` and read order PII. Judged **acceptable** for a single-operator localhost-only tool (PR-03 guarantees the operator surface never deploys off localhost; the spend/state-mutating admin routes DO require a session — the right asymmetry) and documented in `AdminBookReview.tsx`. **Flips to BLOCKING the instant the operator surface becomes multi-user or network-exposed** — the fix then is admin-scoped, session-gated variants of those four routes.
