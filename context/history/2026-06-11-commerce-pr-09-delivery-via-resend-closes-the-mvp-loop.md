## 2026-06-11 — Commerce PR-09: Delivery via Resend (closes the MVP loop)

**Branch:** `feature/delivery-resend` → `main` (`1dc0ea4`, merge `9b354c5`) · Phase 4 (nextjs-ui-builder)

Commerce build, PR-09 — **Phase 4, the last MVP PR. Closes the end-to-end loop: order → pay → generate → approve → email → download.** On the operator's Approve, the order mints an unguessable delivery token, persists it, emails the customer a *Quietly Kept*-tone download link, and moves `approved → delivered`; the link goes to a **public** `/download/[token]` page that serves the order's PDF via a short-lived signed Supabase URL. Reuses the whole approve/render stack unchanged; the new surface is the delivery layer + the public download page/route + the `approved → delivered` transition. Both products (Story-1 book / Story-2 letter) deliver identically.

- **`lib/delivery/` (new commerce layer).** `token.ts` is **pure** (`node:crypto`): `mintDeliveryToken()` → 256-bit base64url (URL-path-safe, the only randomness) + `isWellFormedToken()` (cheap length+charset reject before any DB hit; fail-closed on non-string inputs via RegExp coercion). `email.ts` is **server-only + operator-only** (lazy `getResend()` mirroring `getOpenAI()`, reads `RESEND_API_KEY`/`FROM_EMAIL`): a pure `buildDeliveryEmail()` → `{ subject, html, text }` (warm grief-appropriate tone, correct noun per `storyType`, `petName` HTML-escaped, `/policies` footer link, **never embeds a raw storage URL**) + a thin `sendDeliveryEmail()`. Added `resend@^6.12.4` (roadmap-pre-approved).
- **Email-on-approve chain (`app/(operator)/api/admin/approve/route.ts`).** After the existing render → `putPdf` → `updateOrderStatus(…, "approved", { pdfKey })`: mint token → `setOrderDeliveryToken` (status stays `approved`) → build the absolute link from **`PUBLIC_SITE_URL`** → `sendDeliveryEmail` → `updateOrderStatus(…, "delivered")`. **The load-bearing design call:** any failure below `approved` (no `PUBLIC_SITE_URL`, token-persist fails, or the send throws) **leaves the order at `approved`** with `{ ok:true, status:"approved", delivery:"failed" }` — a finished book is never falsely advanced to terminal `delivered`. Token is minted+persisted **before** the email so the link is valid the moment it's sent. `PUBLIC_SITE_URL` is read from env (not the request origin) because Approve runs on operator/localhost — deriving the origin would email a useless `localhost` link in production. The chain runs at most once per order (the `status !== "awaiting_review"` 409 gate); a future re-send (out of scope) should reuse the stored token.
- **Public download page + API route (the boundary-critical split).** The download **page** (`app/(public)/download/[token]/page.tsx`) is a **client-safe** `"use client"` page (fetch-on-mount, mirrors `/create/preview`) that renders the "ready / download" state or a **soft expired/invalid notice** (no white-screen) — it must NOT import `lib/supabase/server` (the `PUBLIC_ENTRIES` ban), so **all DB + signing lives in the API route**. `app/(public)/api/download/[token]/route.ts` is a **public API route** (PUBLIC_API_ENTRIES — service-role allowed, engine-free, **no `assertOperator()`**): `isWellFormedToken` → `getOrderByDeliveryToken` → mint a **5-min** signed URL via `signedPdfUrl` with the correct download filename (`getStory(storyType).pdfFilename(inputs)` — registry-driven, product-agnostic) → `{ ok, downloadUrl, filename, petName }`. **No-enumeration:** malformed / unknown / found-but-no-`pdfKey` all return an **identical** `404 invalid_or_expired` (no oracle).
- **Store (`lib/order/store.ts`):** `setOrderDeliveryToken(id, token)` (status-agnostic patch mirroring `setOrderLsId`) + `getOrderByDeliveryToken(token)` (validates shape first → `null` on no-match; the only lookup the public route reaches — by opaque token, never an order id, so no IDOR). Index-backed by migration 0001's `orders_delivery_token_key`. `lib/supabase/storage.ts`: `signedPdfUrl` gained an optional `downloadName` (sets `createSignedUrl`'s `download` option — verified against the installed SDK).
- **Refund/remake policy — finalized (the last deferred MVP PM decision, PM-approved).** `app/(public)/policies/page.tsx` Refunds & remakes section is now real copy (30-day window → free remake → full refund if still not right, no-justification tone); the email + download page both link here.
