## 2026-06-10 — Commerce PR-03: Public/Operator Split + Env Gate

**Branch:** `feature/public-operator-split` → `main` (`dfe79e8`, merge `755fdc5`) · Commerce Foundation (nextjs-ui-builder)

Commerce build, PR-03 — **the security spine**. A behaviour-preserving structural refactor that splits the app into a **public** surface (safe to deploy to Vercel) and an **operator** surface (runs locally, holds the OpenAI key + Puppeteer engine + the service-role Supabase client + soon the admin queue). The single most important security boundary in the whole commerce build: a regression here would leak the key path to a public host. No new feature — the local wizard works end-to-end exactly as before, same URLs, just relocated. Mirrors the feature-14 keystone-refactor discipline (structure changes, behaviour preserved, all tests green) and reuses the established "no engine in the client/public graph" boundary technique (`disk.ts` server-only imports, the feature-19 `lib/pdf/filename.ts` split, the PR-02 client-safe catalog).

- **Route groups (URLs unchanged — route groups are URL-transparent).** `app/(public)/` holds the always-on Vercel surface (landing `/` + the story picker; storefront/order/delivery land in later PRs). `app/(operator)/` holds the engine: the **8 engine-touching API routes** (`generate-illustrations` [GET+POST], `regenerate-illustration`, `render-pdf`, `preview`, `update-text`, `session`, `upload`, `test-ai/generate-reference`) + all 11 `create/*` wizard pages (incl. `generate`/`preview`, reused by the operator review UI later) + the `test-ai` dev scaffold. All moves are **pure git renames** (no logic change); colocated `route.test.ts` files moved alongside their routes with identical `./route` imports.
- **`lib/runtime/surface.ts` — the env gate.** Reads `DEPLOY_TARGET` (`public` | `operator`); exports `deployTarget()`/`isOperator()`/`isPublic()`/`assertOperator()`. **Default = `operator`** (fail-closed): only the exact string `"public"` locks down, so a forgotten env on a deploy can only ever make the public surface *more* restrictive, never widen exposure; local `npm run dev` keeps today's full app. Operator API routes call `assertOperator()` as the **first statement of every verb** (returns a plain-text 404 `Response`, indistinguishable from a non-existent endpoint — `notFound()` is for page segments, not Route Handlers).
- **The operator page gate is per-request (QA-hardened).** `app/(operator)/layout.tsx` (new) `notFound()`s the whole operator page group under public. QA found the page gate was originally **build-time-baked** (the statically-prerendered `/create/*` froze the `notFound()` at build), so an app *built* in operator mode then served with `DEPLOY_TARGET=public` only at runtime would serve the inert wizard HTML as 200 (no key/engine leak — those are the per-request API routes, which 404 unconditionally). Closed by adding `export const dynamic = "force-dynamic"` to the operator layout so the page gate evaluates per-request, **build-env-independent** — verified the exact scenario flipped 200 → 404. Consequence: `(operator)` pages are now dynamic (`ƒ`); the `(public)` landing `/` stays static (`○`).
- **Build-time boundary assertion (the load-bearing guard).** `lib/runtime/surface.boundary.test.ts` statically walks the public import closure (`app/layout.tsx` + `app/(public)/page.tsx`, resolving `@/` + relative + dynamic imports) and asserts it contains **no** engine module (`lib/ai/client|generate|prompts`, `lib/pdf/render|template`, `lib/supabase/server`, `lib/session/disk`) and **no** engine package (`puppeteer`, `openai`, `@supabase/supabase-js`). Proven **non-vacuous** (injecting an engine import made it fail on both the module and the transitive package; reverted) and carries a sanity floor so a silent resolution failure can't pass it trivially. The real public closure = 8 modules, only `next` as a third-party dep. Plus `all-operator-routes-gate.test.ts`: a **table-driven** test that every operator verb 404s under public (asserting the 404 came from the gate, not a side effect — `fs`/`getOpenAI`/`puppeteer` mocked, $0, nothing written) **and a drift guard** that walks `app/(operator)/api/` on disk and fails if a route isn't registered in the table — so a future operator route added *without* the gate breaks the suite.
- **Key-path confinement (verified three ways).** `OPENAI_API_KEY` (`lib/ai/client.ts`, lazy read) and `SUPABASE_SERVICE_ROLE_KEY` (`lib/supabase/server.ts`) remain reachable only from operator paths. Confirmed by the import-closure test, an independent re-trace of the public closure, and a `grep` of the public `.next/static` build (0 occurrences of the key names / `puppeteer` / `getSupabaseAdmin` / `renderStoryPdf`).
- **`vercel.json`** sets `DEPLOY_TARGET=public` for build + runtime (build-time so the prerender gating is correct on Vercel; runtime so the per-request gates fire). `.env.local.example`, `README` (structure + a "Deploy — public vs. operator surface" section), and `coding-standards.md` (route-handler location now `app/(operator|public)/api/*`; a new *Deploy-surface boundary* entry in the commerce-layer map; `DEPLOY_TARGET` recorded as non-secret runtime config) all updated on-branch.
