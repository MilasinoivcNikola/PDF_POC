Milestone 4, feature 08 — **first half of Milestone 4**. Built the six-step browser wizard that captures every Story-1 input, persists progress across refreshes, and on Generate writes a complete `StorySession` to disk for the pipeline (features 07/09) to pick up. Replaced the wizard-page + component stubs and the `/api/session` 501 stub; added the Context provider, the draft→session bridge, and a server-only disk module. Faithful to `prototypes/wizard.html` (warm editorial, single column, sticky section heading, "Step NN of 06", "Saved just now").

- **State — `components/wizard/WizardProvider.tsx`** (`"use client"`, mounted once in `app/create/layout.tsx`): holds a `StoryDraft`, hydrates from `loadDraft()`/`newDraft()` in a mount effect (SSR-safe, no hydration mismatch), and every field change shallow-merges → `saveDraft()` (localStorage key `quietly-kept:draft`) → bumps `savedAt`. Context value `useMemo`'d with `useCallback` updaters. Refresh preserves values **and the same session `id`**. Context + localStorage only — no state lib, per plan.
- **Steps — `app/create/{upload,pet,child,memories,style,generate}/page.tsx`**: one concern per route, labeled 1–6, generate = 06 (the plan's structure, **not** the prototype's folded "Step 02 of 06" pet screen — flagged as the intended decision). Collects every merge field + all three toggles (`deathType`/`beliefFrame`/`otherPetsInHome`). Generate assembles via `draftToSession`, POSTs to `/api/session`, swaps to a "saved" hand-off state (the live progress animation is feature 09).
- **Components — `StepShell` / `ProgressBar` / `ImageUploader`**: shared chrome (header, sticky two-column heading, footer Back/Continue, save status), per-step `.steps__bar` fill via a `--steps-remaining` CSS var, and a drag-drop + file-picker uploader with preview + replace + a gentle non-blocking low-res notice (short edge < ~512px). Upload POSTs to `/api/upload` passing `draft.id` as `sessionId`, so the photo lands under `./uploads/[id]/` tied to the same id the session writes under; returned `path` → `pet.photo`.
- **Persistence — `app/api/session/route.ts`** (was 501): `POST` validates the required fields server-side behind the `isSafeSessionId` traversal guard, calls `writeSession()`, returns the house shape (`{ ok:true, id }` / `{ ok:false, error:"snake_case" }`). **Module-boundary fix:** the client provider became the first client importer of `lib/session/storage.ts`, and webpack resolves even dynamic `node:` imports — so server-only `writeSession`/`readSession` were split into **`lib/session/disk.ts`** (static `node:` imports, never client-reachable), leaving `storage.ts` client-safe. Confirmed by `/create/*` prerendering statically with no `fs` in the bundle.
- **`lib/session/draft.ts`** — pure, testable bridge: `missingRequiredFields` / `isDraftComplete` / `draftToSession` (assembles a `StorySession`, trims names, fills skipped enum/toggle optionals with master-template defaults, drops empty `parentDedication`, `images: []`, throws `missing_required_fields` on any gap).
- **CSS** — ported the wizard-only classes that lived **inline in `wizard.html`'s `<style>`** (`.wizard*`, `.form-layout`, `.wizard-footer`, `.save-status`, `.photo-preview`, `.style-grid`, `.style-card`) into the single-sourced `app/globals.css`, token-based; added on-brand `.notice` / `.notice--required`. Reused the already-ported `.field` / `.radio-option` / `.upload-zone` / `.steps`.

Nikola's call: **make the four free-text fields required too** (`breedColor`, `favoriteActivity`, `sleepingSpot`, `favoriteMemory`), so the required set is **7** and a written session is always renderable. Verified end-to-end: a 7-field-only session now flows through `resolveStory` with no `MergeError`.
