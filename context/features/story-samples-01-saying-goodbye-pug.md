# Story Samples PR-01: "Saying Goodbye" (Pug) Sample Set — refresh Story 1 onto the standard harness

> **Type:** Feature (single PR). **Status:** Spec — awaiting go-ahead to implement.
> **Surface:** sample assets + one fixture + the worked-example content map + docs.
> No engine, route, layout, or pricing change.

---

## Goal

Replace Story 1's current placeholder sample (**Bo the boxer / Liam**, generated by
the one-time flagship HIGH run) with a **real photo-driven example** painted from the
committed `uploads/sample-photos/story-1.jpg` (a **fawn pug** with a young girl in a
park). In doing so, **graduate Story 1 off the flagship-HIGH exception** and onto the
same standard mixed-`PRODUCTION_QUALITY` harness (`proto:sample` → `sample-capture`)
that all 7 other titles use — so the Story-1 sample finally shows *what a paying
customer actually receives* (~$1/book), not a $3 all-HIGH outlier.

Decisions locked with the PM (2026-06-17):
- **Replace** the Bo sample (one sample/example per title stays the rule).
- **Standard mixed tier** (~$1), not the flagship HIGH (~$3).

## The source photo

`uploads/sample-photos/story-1.jpg` — 6016×4016, 300 DPI, ~4.7 MB (it is already a
tracked file under the `uploads/sample-photos/` `.gitignore` negation, so the sample
is reproducible from a clean checkout — the PR-0 pattern; this replaces the old
gitignored boxer path `uploads/high-run-candidates/test-image.jpg`).

Subject: a **fawn pug** — classic black mask over the muzzle/ears, deep velvety brow
wrinkles, big dark round eyes, tightly curled tail — standing on a low park wall beside
a **young girl (~7–8)** who is smiling and resting her hand on his back. Black iron
fence, green grass, warm evening light.

Two notes that make this photo unlike the other 7 sample inputs:
- **It is the catalog's third dog** (after the Story-1 boxer it replaces and the
  Story-5 senior dog). A pug's silhouette is distinct enough that this is not a
  consistency concern.
- **It contains a child as well as the pet.** This is *good* for Story 1 (the cover and
  several pages show the child anyway), and the reference step is pet-only by
  construction — `buildReferencePrompt` paints *"the pet in the reference photo…
  maintain the pet's exact appearance"* keyed off `breedColor`, so the girl should not
  leak into the locked reference. We keep a QA gate (below) to confirm. Per the master
  template the **child stays stylized** (3/4 / from-behind) — we do **not** reproduce
  the real girl's face.

## Build / run recipe (the cost step is manual, PM-gated)

```
# 1. (after the fixture lands) PAID — generate at the locked mixed PRODUCTION_QUALITY:
npm run proto:sample fixtures/sample-story1-dog.json        # ≈ $1, several minutes

# 1a. QA GATE — before trusting the run, eyeball generated/sample-story1-dog/reference.png
#     + cover.png: is the PUG on-model and the child absent from the reference?
#     (cheap insurance given the child-in-photo input). Re-run if drifted.

# 2. $0 — capture committed slim web assets into public/samples/story-1-book/:
tsx --tsconfig scripts/tsconfig.json scripts/sample-capture.ts \
  --id sample-story1-dog --out story-1-book

# 3. wire book-questions.ts + test pin + docs; npm run test:run && npm run build
```

`generated/` + `output/` are gitignored; only the `public/samples/story-1-book/` JPGs +
`preview.pdf` + `source-photo.jpg` and the `fixtures/` JSON are committed.

  "See the full book (PDF)" link 200s on the new slim `preview.pdf`, the

  "the example we used" answers read as Mango/Maya. No 404s.

## Risks & mitigations

- **Child-in-reference leak** → QA gate at step 1a; reference prompt is pet-only by
  construction. Low risk, cheap to recheck on a $1 run.
- **Pug on-model across 13 scenes** → pugs have strong, distinctive markings (mask +
  wrinkles + curled tail); Approach A anchors well. Eyeball the climax/bookend pages.
- **Story 1 visibly loses fidelity** vs the old all-HIGH sample → intended: the new
  sample is *honest* about delivered quality, and cover+page-12 still render HIGH.
